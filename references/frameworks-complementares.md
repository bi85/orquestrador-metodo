# Frameworks Complementares — Criador de Método Próprio

Arquivo de referência para aprofundamento nos módulos de Nomeação, Pitch e Portfólio.

## NOMEAÇÃO DO MÉTODO — Expert Secrets (Russell Brunson)

Brunson chama de "Framework Proprietário" (Proprietary Framework) o coração de
qualquer oferta de especialista. O método precisa de:

### 1. Um Nome Proprietário
Não pode ser "Metodologia de Vendas". Precisa ser algo que só você tem.
Razão: o nome cria um novo "veículo" — uma nova categoria que o cliente não
pode comparar com o que já existe. Comparação mata o preço.

**Critérios para um bom nome:**
- É diferente do que todos já usam na área
- Comunica a transformação ou o processo único
- É fácil de lembrar e repetir
- O criador se sente orgulho de dizer em voz alta

**Exemplos de estruturas de nome:**
- Siglas: AIDA (Atenção, Interesse, Desejo, Ação), SPIN Selling, DCR (Destino, Caminho, Rota)
- Metáforas: Método Waze, Framework Bússola, Sistema Âncora
- Nome + Processo: Método [Nome do criador], Framework [Palavra que evoca resultado]
- Adjetivo + Substantivo: Pitch Perfeito, Venda Consultiva, Diagnóstico Cirúrgico

### 2. Uma História de Origem (Origin Story)
Antes de apresentar o método, conte a história de como você descobriu
o problema e criou a solução. Estrutura:
- Eu estava em [SITUAÇÃO DIFÍCIL]
- Tentei [MÉTODO CONVENCIONAL] e não funcionou
- Descobri [INSIGHT CHAVE]
- Criei [NOME DO MÉTODO]
- Resultado: [TRANSFORMAÇÃO]

Isso cria legitimidade sem precisar de credenciais acadêmicas.

### 3. Uma Causa Futura
O método não é só um produto — é um movimento. O que você acredita que
mudaria no mundo/mercado se mais pessoas usassem seu método?
Essa causa atrai seguidores e diferencia de concorrentes.

## POSICIONAMENTO — StoryBrand (Donald Miller)

### O Cliente é o Herói. Você é o Guia.

Erro comum: o criador de método se posiciona como herói da história
("Eu fiz X, conquistei Y, sou especialista em Z").

O correto: o CLIENTE é o herói. Você é o Guia que aparece com um método
para ajudá-lo a vencer o desafio.

### O Framework SB7 Aplicado ao Método

| Elemento | Pergunta | Resposta para o método |
|----------|----------|----------------------|
| Herói | Quem é o cliente? | [Perfil definido no Briefing] |
| Problema | Qual é a dor? | Externo / Interno / Filosófico |
| Guia | Por que você? | Empatia + Autoridade |
| Plano | Como você resolve? | O Caminho (passos do método) |
| CTA | O que o cliente faz? | Próxima ação clara |
| Sucesso | Como é a vida depois? | O Ponto B do Destino |
| Fracasso | O que ele perde se não agir? | Custo da inação |

### Os 3 Níveis de Problema (Miller)
Todo cliente tem 3 camadas de dor. O método resolve todas:

**Externo (visível):** O problema concreto que ele descreve.
Ex: "Não consigo organizar meu conhecimento para vender."

**Interno (emocional):** O que ele sente por causa do problema.
Ex: "Sinto que estou desperdiçando o que sei."

**Filosófico (identidade):** O que isso diz sobre quem ele é.
Ex: "Pessoas como eu não conseguem escalar."

O pitch mais poderoso nomeia os 3 níveis.

## PITCH — Pitch Anything (Oren Klaff)

### O Frame Controls Everything

Klaff ensina que o pitch falha quando o "frame" (enquadramento mental)
do comprador domina o do vendedor. Para manter o frame:

**1. Apresente novidade, não familiaridade**
O cérebro compra o que é novo. Não diga "é como X". Diga o que é diferente.
"Isso não é um curso de vendas. É um método de extração de método."

**2. Crie tensão de tempo (não urgência falsa)**
"Estou abrindo apenas para [X] pessoas neste ciclo porque a qualidade
do acompanhamento não escala além disso."

**3. Use o Frame de Prêmio (Prize Frame)**
Você não está pedindo para o cliente comprar. Você está qualificando
se ele é a pessoa certa para o método.
"Esse método funciona para quem já tem experiência e quer organizá-la.
Não é para quem está começando do zero."

### Estrutura do Pitch em 4 Minutos (adaptada)

**1. O Frame (30 seg):** Estabeleça quem você é e por que está aqui.
"Trabalho com [ÁREA] há [X anos] e nesse tempo descobri que..."

**2. O Problema (60 seg):** Nomeie a dor com precisão cirúrgica.
"O maior problema de quem sabe muito é [PROBLEMA EXTERNO].
Por baixo disso, o que realmente dói é [PROBLEMA INTERNO]."

**3. O Método (90 seg):** Apresente o caminho em 3 passos, não mais.
"A solução que desenvolvi tem 3 etapas: [P1], [P2], [P3].
Cada uma resolve uma parte do problema..."

**4. A Oferta (60 seg):** Produto, preço, resultado esperado, CTA.
"Para quem se qualifica, ofereço [PRODUTO] que entrega [RESULTADO]
em [PRAZO]. O investimento é [VALOR]. Próximo passo: [CTA]."

## PORTFÓLIO E ESCALA — E-Myth (Michael Gerber)

### O Problema do Especialista

Gerber identifica o "Mito do Empreendedor": o técnico brilhante que
não consegue escalar porque o conhecimento está preso nele.

A solução é o que Gerber chama de "Sistema de Entrega":
transformar o que o especialista faz em um processo que outros
(ou outros formatos) podem executar.

### Aplicação ao Método Proprietário

| Nível de Dependência | Formato | Escalabilidade |
|---------------------|---------|----------------|
| Total dependência | Consultoria 1:1 | Baixa — vende tempo |
| Alta dependência | Mentoria individual | Média — limitado por agenda |
| Média dependência | Mentoria em grupo | Alta — mesma hora, múltiplos |
| Baixa dependência | Curso gravado | Muito alta — vende dormindo |
| Zero dependência | Material (livro/e-book) | Total — mas ticket baixo |

**Estratégia recomendada:**
Comece com alta dependência (consultoria/mentoria) para validar o método
com clientes reais. Depois sistematize o que funcionou em formatos
com menor dependência. Nunca o contrário.

### A Regra de Ouro do Portfólio

Mantenha no máximo 3 produtos ativos simultaneamente:
1. **Entrada:** Para capturar audiência (baixo ticket)
2. **Core:** Produto principal (ticket médio-alto)
3. **Premium:** Para quem quer mais resultado (ticket alto)

Mais que isso divide atenção e confunde o cliente.

## VALIDAÇÃO — Lean Startup (Eric Ries) Aplicado a Métodos

Antes de criar tudo, valide com o menor esforço possível.

### MVP do Método (Produto Mínimo Viável)

**Nível 1 — A Conversa (gratuito):**
Ofereça uma sessão de diagnóstico ou consultoria experimental.
Objetivo: validar se o problema que você identificou realmente dói.

**Nível 2 — A Degustação (gratuito ou baixo custo):**
Entregue uma parte do método para um grupo seleto.
Objetivo: validar se o Caminho resolve o problema como esperado.

**Nível 3 — O Piloto (preço real):**
Venda a primeira turma/cliente do método completo.
Objetivo: validar se as pessoas pagam o preço que você imaginou.

**Regra:** Só construa o produto completo (plataforma, gravações, materiais)
depois de ter pessoas pagando pelo piloto.

### Métricas de Validação

| O que medir | Sinal positivo | Sinal de ajuste |
|------------|----------------|-----------------|
| Taxa de resposta ao pitch | > 30% demonstram interesse | < 10% — reveja o problema ou o público |
| Conversão da degustação | > 20% pedem mais | < 10% — reveja a didática |
| Pagamento pelo piloto | 3+ pessoas pagam | 0 pagamentos — reveja preço ou formato |
| Resultado do cliente | Atingiu o Ponto B | Não atingiu — reveja o Caminho |
